Automation Data Engineering GIS

Automatización ETLT para generación de proyectos QGZ de elegibilidad forestal

La generación manual de proyectos QGZ era costosa en tiempo, heterogénea en formato y difícil de encajar posteriormente en base de datos corporativa.

Diseñé una ETLT completa para obtener información vía APIs, normalizarla, registrarla con logs y generar proyectos geoespaciales listos para explotación interna.

La solución se contenerizó con Docker y se orquestó con Prefect 3 mediante tasks y flows. El servidor central aloja Prefect Server, su UI y la base de datos propia del sistema; los equipos en red local actúan como workers que escuchan y ejecutan cargas distribuidas conectándose por puertos y servicios internos, permitiendo separar control, ejecución y persistencia de datos de forma robusta.

Python SQL PostgreSQL Docker Prefect 3 APIs
Resultado: Reducción del tiempo de generación de ~30 minutos a ~2 minutos, con mayor normalización, trazabilidad e integración en sistema corporativo.

ML Remote Sensing

Modelo de ML para predicción de usos elegibles y no elegibles para reforestación

El análisis técnico detallado de grandes superficies consume tiempo, especialmente en fases previas a negociación, donde se necesita una estimación preliminar rápida y no un estudio exhaustivo desde el primer momento.

Diseñé dos versiones del modelo con Random Forest y XGBoost para clasificar áreas arboladas, no arboladas pero potencialmente elegibles y no elegibles, usando imágenes Sentinel-2 recientes.

Python R XGBoost Random Forest Sentinel-2
Resultado: Análisis preliminares en segundos para priorizar zonas y acelerar evaluación inicial de oportunidades.
