Automatización Data Engineering GIS

Automatización de generación de proyectos QGZ para elegibilidad de reforestación

Se automatizó un flujo ETLT completo para generar proyectos QGZ listos para estudio técnico, extrayendo información vía APIs, normalizando datos y registrando logs de ejecución para trazabilidad.

La arquitectura se desplegó con contenedores, con un servidor central que aloja base de datos y orquestación, y nodos de trabajo conectados en LAN mediante workers que ejecutan tareas distribuidas.

  • Stack: Python, SQL, PostgreSQL/PostGIS, Docker, Prefect
  • Orquestación: flows, tasks, worker pools, Prefect Server UI y ejecución distribuida
  • Resultado: generación de proyecto de ~30 min a ~2 min
Machine Learning Remote Sensing Sentinel-2

Modelo de ML para clasificar zonas arboladas, no arboladas elegibles y no elegibles

Diseño de modelos preliminares para acelerar el análisis de grandes superficies en fases tempranas de negociación y estudio. Se desarrollaron aproximaciones con Random Forest y XGBoost a partir de imágenes Sentinel-2 recientes.

  • Stack: Python, R, XGBoost, Random Forest
  • Objetivo: reducir trabajo manual en análisis preliminares
  • Resultado: evaluación inicial de terrenos en segundos

QGIS Plug-ins Productividad

Desarrollo de plug-ins y toolbox para QGIS

Construcción de utilidades para encapsular procesos repetitivos y costosos para técnicos GIS. Entre las funcionalidades más relevantes: generación automática de mapas, extracción de listados catastrales mediante operaciones espaciales y subida normalizada de capas a base de datos con un solo clic.

  • Generación automática de mapas con cuadros informativos desde proyecto + BDD
  • Listados de parcelas por intersección, diferencia y otras operaciones espaciales
  • Subida de capas a BDD con normalización de nombres y tipos
